# Start A Pallinky Change

1. Inspect the repository, current Git state, and relevant documentation.
2. Confirm one user outcome and observable definition of done.
3. Interview the operator one question at a time when product decisions are
   unclear; recommend an answer in plain English.
4. Use relevant Superpowers skills throughout. Prefer brainstorming or planning
   for unclear work, systematic-debugging for bugs, test-driven-development
   where practical, and verification-before-completion before finishing.
5. Confirm the work is isolated on an appropriate task branch or Worktree.
6. Implement and verify the change end to end, following `AGENTS.md`.

Do not broaden the task without operator approval.
