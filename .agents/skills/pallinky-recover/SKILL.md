# Recover Pallinky Work

1. Inspect Git status, branch, commits, worktrees, and relevant task context.
2. Explain in plain English what is saved, unsaved, verified, unverified, and
   failing.
3. Use Superpowers systematic-debugging when behavior or checks are failing,
   and verification-before-completion before declaring recovery complete.
4. Recommend the safest next action that preserves all current work.
5. Explain impact before changing branches, rebasing, reverting, or restoring.
6. Follow `AGENTS.md` approval rules. Never discard work, push, force-push,
   merge, access production, or trigger a release without the required approval.
